# 竞赛规则深度解读

## 一、得分机制详解

### 1.1 单条序列得分公式

```
相对亮度得分 = Finitial / Finitial_WT
热稳定性得分 = Ffinal / Finitial
综合得分     = 相对亮度得分 × 热稳定性得分
             = (Finitial / Finitial_WT) × (Ffinal / Finitial)
             = Ffinal / Finitial_WT
```

**关键洞察**：综合得分在数学上等价于 **"热处理后亮度相对于WT初始亮度的比值"**。

但这并不意味着只需追求热稳定性！因为：
- 若 Finitial < 0.3×Finitial_WT → **该序列直接淘汰，两项均记0分**
- 因此必须确保亮度底线，再最大化热稳定性

### 1.2 淘汰机制

```
if Finitial < 0.3 × Finitial_WT:
    相对亮度得分 = 0
    热稳定性得分 = 0
    综合得分     = 0
    状态         = "淘汰"
else:
    正常计算得分
```

**策略含义**：
- 保守序列（Seq-1/2）确保不跌破30%线
- 激进序列（Seq-4/6）即使亮度较低，也需保持在30%以上
- 我们的设计预估亮度均 > 75% WT，安全垫充足

### 1.3 批次对照

> "每批包含WT对照"

这意味着：
- 你的序列与**同批次**WT比较，而非绝对标准
- WT的Finitial和Ffinal作为分母
- 批次效应（不同时间、不同操作者）被WT对照消除

**策略含义**：无需担心批次间差异，只需相对WT表现好即可。

## 二、奖项规则解读

### 2.1 金奖（前30%）

**排名依据**：6条序列中得分最高的那条（Best Top-1）

**策略**：
- 不需要6条都高分，只需1条冲击前30%
- 因此可以承担高风险设计（如Seq-6），只要其他序列保底

### 2.2 银奖（其余若干比例）

未获金奖的队伍中，按Top-1成绩排名分配。

### 2.3 独立荣誉奖

- **最佳亮度奖**：Finitial/Finitial_WT 最高的序列
- **最佳热稳定奖**：Ffinal/Finitial 最高的序列

**策略**：
- 即使综合得分不顶尖，单项突出也可获奖
- Seq-5可冲亮度奖，Seq-6可冲热稳定奖

## 三、提交规则详解

### 3.1 序列文件 (.csv)

#### 长度限制：220-250 aa
- sfGFP为238 aa，正好在中间
- 若需调整，可在N端/C端适度截短/延伸
- 但**不建议大幅改变长度**，可能破坏折叠

#### 字符规范
- 必须以 **M** 开头（起始密码子ATG翻译为Met）
- 仅20种标准氨基酸大写字母
- 严禁包含终止密码子（*）或标点符号

**常见错误**：
- 使用小写字母 → 格式错误
- 包含空格或换行 → 格式错误
- 以其他氨基酸开头 → 格式错误

#### 母序列选择
> "参赛序列可从零设计，也可基于任意 GFP 家族参考序列优化，不要求以 sfGFP 为母序列"

**解读**：
- 可以使用 eqFP578、cgreGFP、avGFP 等其他GFP家族成员
- 但sfGFP在CFPS中表现最优，强烈建议作为骨架

#### 排除列表
> "请在提交前逐条与Exclusion_List.csv中的序列做完全一致性比对"

**解读**：
- **完全一致**才排除，单个氨基酸差异即视为不同序列
- 排除列表通常包含：WT序列、常见变体（如EGFP、YFP等）
- 若冲突，微调突变即可（如E222Q → E222G）

### 3.2 设计思路文档 (.pdf)

#### 内容要求
1. **完整流程**：从零开始或组合现有模型的生成流程
2. **算法管线**：如何平衡亮度和稳定性目标
3. **筛选逻辑**：最终选择这6条序列的原因
4. **LLM Agent**：如使用，展示逻辑树和关键执行日志

#### 评审权重
> "在客观得分相近的情况下，优先选择思路清晰、技术路径前沿且开源的队伍进行宣传和展示"

**解读**：
- 得分是硬指标，但文档质量影响"宣传展示"机会
- 清晰的技术路径和开源精神是加分项

### 3.3 开源仓库链接

#### 要求
- 公开可访问（GitHub/GitLab/HuggingFace等）
- 包含README.md，说明环境配置、模型依赖、推理代码运行方式

#### 建议
- 使用GitHub（最常用）
- 包含LICENSE（MIT推荐）
- 包含requirements.txt
- 代码有注释，关键参数说明

## 四、验证流程解读

### 4.1 实验流程

```
1. 序列格式验证
   └─→ 格式正确？不在排除列表？
       └─→ 通过 → 进入合成
       └─→ 失败 → 退回

2. DNA模板合成
   └─→ 使用DNAChisel反向翻译
   └─→ 统一添加启动子、RBS、终止子
   └─→ 标准化浓度

3. Cell-Free蛋白合成 (CFPS)
   └─→ 无细胞体系表达
   └─→ 无分子伴侣辅助

4. 初始亮度测量 (Finitial)
   └─→ 测量折叠后的荧光强度

5. 热处理 (72°C)
   └─→ 高温变性挑战

6. 热后亮度测量 (Ffinal)
   └─→ 测量恢复后的荧光强度
```

### 4.2 CFPS体系特点

> "运用计算方法设计兼具高荧光亮度和优良热稳定性的GFP变体"

**关键约束**：
- **无分子伴侣**：蛋白质必须能自发正确折叠
- **无翻译后修饰**：依赖序列本身的折叠信号
- **快速表达**：通常几小时完成

**策略含义**：
- sfGFP的"superfolder"特性完美适配
- 避免引入需要伴侣协助的突变
- 表面电荷优化（Seq-5）可增强溶解度

## 五、常见策略误区

### 误区1："亮度不重要，只要热稳定性高"

**错误**：若Finitial < 0.3×WT，序列直接淘汰，热稳定性再高也没用。

**正确**：亮度是底线，热稳定性是上限，两者相乘才是综合得分。

### 误区2："突变越多越好"

**错误**：突变数增加会累积风险，可能导致折叠失败。

**正确**：精准突变 > 大量突变。我们的设计从1个到5个突变梯度分布。

### 误区3："必须用最新AI模型"

**错误**：ESM-3、AlphaFold3等新模型未必比经典方法更适合此任务。

**正确**：模型选择应匹配任务需求。ESM-2足够用于突变筛选，Rosetta用于稳定性精算。

### 误区4："排除列表只是形式"

**错误**：若序列在排除列表中，直接取消资格。

**正确**：必须逐条比对，完全一致才排除，稍有差异即可。

### 误区5："设计文档不重要"

**错误**：文档不影响客观得分，但影响宣传展示机会。

**正确**：在得分相近时，文档质量决定额外荣誉。

## 六、策略优化建议

### 6.1 针对得分公式

由于综合得分 = Ffinal/Finitial_WT，理论上：
- 最大化 Ffinal（热处理后亮度）
- 不直接依赖 Finitial，但受30%约束

**最优策略**：
1. 确保 Finitial > 0.5×WT（安全垫）
2. 最大化 Ffinal/Finitial（热处理恢复率）
3. 这等价于最大化蛋白质的可逆折叠能力

### 6.2 针对独立奖项

| 奖项 | 优化目标 | 推荐序列 |
|------|---------|---------|
| 最佳亮度 | Finitial/Finitial_WT | Seq-5 (CFPS优化) |
| 最佳热稳定 | Ffinal/Finitial | Seq-6 (探索型) |

### 6.3 针对金奖排名

金奖依据Top-1成绩，因此：
- 6条序列中至少有1条冲击高分
- 保守序列保底，激进序列冲奖
- 我们的分布：2保守 + 2中等 + 2激进，合理
